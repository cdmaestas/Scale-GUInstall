Important Disclaimer

This is a web frontend for the IBM Storage Scale Installation Toolkit (spectrumscale) and is provided as a convenience interface. By proceeding, you acknowledge the following:

  • This tool interacts with your production or test IBM Storage Scale cluster. Incorrect configuration may result in data loss or cluster instability.
  • This frontend does not replace reading the official IBM Storage Scale documentation. Ensure you understand the implications of each operation before executing.
  • Pre-check and post-check options are strongly recommended before and after every installation, deployment, or upgrade operation.
  • This tool requires the IBM Storage Scale Installation Toolkit to be installed and the spectrumscale command to be accessible on the installer node.
  • IBM Storage Scale is a licensed product. Ensure you have appropriate entitlements before proceeding.
  • This frontend is a community/unofficial helper tool and is not an official IBM product or service.

Understanding Installation Toolkit Options

Reference: spectrumscale command options for installing IBM Storage Scale and deploying protocols

spectrumscale commandPurpose
node addAdds node specifications (hostname, roles) to the cluster definition file
node deleteRemoves node specifications from the cluster definition file
nsd addAdds NSD (Network Shared Disk) specifications to the cluster definition file
nsd clear / delete / list / modifyManage NSDs in the cluster definition
filesystem list / modifyList or change block size and mount point of a filesystem
config gpfsAdds IBM Storage Scale–specific properties to the cluster definition file (including --gplbin_dir for prebuilt kernel modules)
config clear gpfsClears a specific property from the GPFS configuration (e.g. --gplbin_dir)
config protocolsProvides CES shared filesystem and environment details for protocol deployment
config perfmonConfigures performance monitoring (zimon/GUI) settings
config populatePopulates cluster definition file from an existing running cluster
installInstalls GPFS packages, creates the cluster, creates NSDs
scaleadmd enableEnables the Scale Admin Daemon on configured nodes (new in 6.0.1)
nodeid defineDefines node identity mapping in the cluster definition (new in 6.0.1)
deployCreates filesystems and deploys protocol services on configured nodes
upgradeUpgrades GPFS and protocol components across the cluster
callhomeConfigures IBM call home (support notification) settings
fileauditloggingConfigures file audit logging settings
nfs_core_dump enableEnables NFS core dump collection on protocol nodes for troubleshooting (new in 6.0.1)

Installation has two stages: (1) use spectrumscale commands to build the cluster definition file, then (2) run spectrumscale install to apply it. Deployment follows the same pattern with spectrumscale deploy. See IBM documentation →

Step 1: Install IBM Storage Scale

Validates prerequisites, installs GPFS packages on all configured nodes, and creates the cluster.

$ sudo /usr/lpp/mmfs/5.x/installer/spectrumscale install --precheck
$ sudo /usr/lpp/mmfs/5.x/installer/spectrumscale install
$ sudo /usr/lpp/mmfs/5.x/installer/spectrumscale install --postcheck

Step 2: Enable Admin Daemon & Node Identity

After install, enable the Scale Admin Daemon on all nodes, then set up TLS-based node identities using X.509 certificates. Required for cluster management in IBM Storage Scale 6.0.1+.

2a — Enable Admin Daemon

$ sudo /usr/lpp/mmfs/5.x/installer/spectrumscale scaleadmd enable

2b — Node Identity (TLS Certificates)

Generates a self-signed CA and per-node X.509 certificates using EC keys, then imports them via scalectl node config set. Follows IBM Storage Scale 6.0.1 node identity documentation.

Cluster Upgrade

Perform an online (rolling) or offline upgrade of the IBM Storage Scale cluster using the installation toolkit.

Target Version

Detected from cluster (run Populate first)
Local directory containing GPFS RPMs or URL of package repository

Upgrade Method

Online (Rolling) Upgrade

Upgrades nodes one at a time while the cluster remains operational. Data access is maintained throughout the upgrade. Suitable for production environments.

Recommended No downtime

Offline Upgrade

Shuts down the entire cluster before upgrading. All nodes are upgraded simultaneously. Faster overall but requires planned maintenance window.

Planned downtime required
Nodes upgraded simultaneously (1 recommended)

Offline upgrade will unmount filesystems and stop all GPFS services

All clients will lose access to the filesystem during the upgrade. Ensure all applications are properly quiesced before proceeding.
